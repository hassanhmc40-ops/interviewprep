# Feature Specification: AI Interview Question Generation via Groq API

## Context & User Stories
**US11 – Generate interview questions**
As a logged-in user, I want to click "Generate interview questions" on a concept detail page to receive 5 realistic technical interview questions generated by the AI API, based strictly on the title and explanation of the concept.

**US12 – View generation history**
As a logged-in user, I want to see all past generations for a concept displayed clearly in chronological order.

**US13 – Delete a generation**
As a logged-in user, I want to delete a set of generated questions that I no longer need.

## Technical Requirements
- **Model & Migration:** `GeneratedQuestion` entity with fields: `id`, `concept_id` (FK), `questions` (JSON array), timestamps.
- **Casting:** Ensure the `questions` attribute is cast to an `array` in the `GeneratedQuestion` model.
- **API Integration:** Call the Groq API endpoint (`https://api.groq.com/openai/v1/chat/completions`) using Laravel's native `Http::` facade. Use the model `llama3-8b-8192` or `mixtral-8x7b-32768`.
- **Prompt Engineering:** Pass a strict system prompt instructing the AI to output exactly 5 technical questions based on the concept title, difficulty level, and user explanation, formatted cleanly.
- **Persistence First:** The parsed API response must be saved to the database as a new `GeneratedQuestion` record *before* returning the view.
- **Error Handling:** Implement robust timeout handling and `try-catch` blocks. If the API fails, display an elegant flash error message.

## 🚫 What I DO NOT want (Negative Constraints)
- DO NOT INSTALL EXTERNAL SDKs OR PACKAGES (e.g., `openai-php/client`). You are strictly required to use Laravel's native `Http::` facade.
- DO NOT commit the Groq API key to version control or hardcode it in classes. It must reside exclusively in `.env` and be loaded via `config('services.groq.key')`.
- DO NOT display AI-generated questions directly to the user without persisting them to the database first.
- DO NOT render a blank page, raw JSON dump, or framework exception screen if the Groq API times out, returns a 500 error, or fails to parse. Catch exceptions and handle them gracefully in the UI.
